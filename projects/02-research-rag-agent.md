# Research / RAG Agent

## 项目目标

构建一个能够检索知识、总结内容并给出引用来源的研究型 Agent。

## 这个项目要证明什么

它主要证明你具备：
- RAG 基础能力
- 检索与生成结合能力
- 结构化总结与 grounded output 能力
- Memory / context management 基础能力

## 最小功能范围

建议至少支持：
- 本地文档或网页内容导入
- 文档切分与索引
- 查询 -> 检索 -> 总结 -> 引用来源 闭环
- 输出结构化摘要
- 多轮查询能力

## 推荐技术栈

- Python
- 向量数据库或轻量检索方案
- LlamaIndex / LangChain / 自己实现最小 RAG
- 嵌入模型 + LLM API

## README 应该写什么

- 数据来源是什么
- 检索流程如何设计
- 如何控制 hallucination
- 如何展示引用来源
- 如何评估检索与总结效果

## 进阶增强项

- reranking
- query rewriting
- 多来源检索
- 长文档分层总结
- 用户偏好记忆

## 面试时如何讲

重点讲：
- 为什么这样切分文档
- 如何选择召回策略
- 如何确保输出有依据
- 如果检索失败，你如何降级处理
