# 12 周学习计划

下面是一份偏工程落地、面向求职的 12 周建议节奏。你可以根据自己的时间压缩或拉长，但最好保持“每个阶段都有项目产出”。

## 第 1~2 周：建立心智模型 + 打通最小闭环

目标：理解什么是 Agent，并亲手完成最小 tool calling 闭环。

建议任务：
- 搞清 assistant、workflow、agent 的区别
- 熟练模型调用与结构化输出
- 手写最小 tool calling 闭环
- 记录基础架构图和设计笔记

## 第 3~5 周：做出第一个 CLI Agent

目标：拥有第一个可运行、可展示的作品。

建议任务：
- 构建单 Agent loop
- 接入至少 3 类工具
- 补日志、重试、状态管理
- 写清 README、使用示例和限制说明

## 第 6~8 周：补强能力层

目标：让 Agent 从“能跑”变成“开始有用”。

建议任务：
- 学 memory、RAG、planning、review loop
- 完成 Research Agent 或 Coding Assistant
- 开始记录评估方式和输出质量标准

## 第 9~10 周：理解 Agent System

目标：从单 Agent 进阶到多 Agent / workflow 视角。

建议任务：
- 学 router、planner-executor、manager-worker 模式
- 研究 Hermes、Claude Code、OpenClaw 等系统设计
- 做一个多 Agent 小项目

## 第 11~12 周：补工程化与求职包装

目标：把项目整理成真正有竞争力的作品集。

建议任务：
- 补 observability、eval、deployment、安全边界
- 打磨 README、项目结构、架构图
- 把项目整理成 GitHub 作品集
- 准备面试时的项目讲解与 tradeoff 表达

## 最终交付目标

12 周结束后，建议你至少拥有：
- 1 个 CLI Tool-Using Agent
- 1 个 RAG / Research Agent 或 Coding Assistant
- 1 个基础 Multi-Agent / Workflow 项目
- 1 套能用于求职表达的 README、项目说明和架构讲解材料
